APIs Description

admin/ - Django admin panel
api/home/ [name='apis-description'] - Home endpoint for API description

api/login/ [name='user-login'] - User login endpoint
- Provide user credentials (username and password) or mobile number (to send OTP) in raw format (JSON)
- If the user doesn't exist and login is done with a mobile number, a user will be automatically created with empty fields
- Example:

{
    "phone_number": "+917601086020"
}

api/validate-otp/ [name='otp-validation'] - Verify OTP endpoint after providing the mobile number in the login endpoint

api/logout/ [name='user-logout'] - User logout endpoint (optional, can be handled client-side)

api/signup/ [name='user-signup'] - User signup endpoint
- Provide credentials like username, email, phone number, and password

api/Restos/ [name='Restos'] - Fetch all restaurant details (permissions: all)

api/products/ [name='products'] - Fetch specific product details of a restaurant based on the resto_id (permissions: authenticated)
- Example: api/products/?resto_id=<resto_id>

api/restaurants/ [name='restaurant-search'] - Search for restaurants using a search bar (permissions: all)
- Example: api/restaurants/?resto_name=<resto_name>
